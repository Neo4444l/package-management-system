================================================================
⚠️ Git 未安装 - 快速解决方案
================================================================

您遇到的错误：
"无法将"git"项识别为 cmdlet、函数、脚本文件或可运行程序的名称"

原因：系统未安装 Git 工具


================================================================
🎯 推荐解决方案（2选1）
================================================================


【方案A】使用 GitHub Desktop ⭐⭐⭐ 强烈推荐！
================================================================

优点：
✅ 无需安装 Git
✅ 无需使用命令行
✅ 图形界面，简单易用
✅ 一键上传代码到GitHub

步骤：
1. 下载 GitHub Desktop
   👉 https://desktop.github.com
   
2. 安装并登录GitHub账号
   （没有账号的话在 https://github.com 注册）

3. 使用图形界面上传代码
   详见：GitHub Desktop部署教程.md

⏱️ 预计时间：5-8分钟
📊 难度：⭐☆☆☆☆（非常简单）


【方案B】安装 Git 命令行工具
================================================================

优点：
✅ 功能强大
✅ 适合开发者

步骤：
1. 下载 Git
   官方：https://git-scm.com/download/win
   镜像：https://registry.npmmirror.com/binary.html?path=git-for-windows/

2. 安装（一路Next即可）

3. 重启命令行窗口

4. 验证安装：
   git --version

详见：Git安装指南.txt

⏱️ 预计时间：5-10分钟
📊 难度：⭐⭐☆☆☆（简单）


================================================================
💡 建议
================================================================

对于新手或不熟悉命令行的用户：
👉 强烈推荐使用 GitHub Desktop（方案A）

对于开发者或熟悉命令行的用户：
👉 可以选择安装 Git（方案B）

❓ 常见问题
================================================================

Q: 必须要用Git吗？
A: 是的，部署到Vercel需要代码在GitHub上。
   但您可以用GitHub Desktop，无需学习Git命令。

Q: GitHub Desktop 和 Git 有什么区别？
A: GitHub Desktop 是图形界面工具，内置了Git
   使用GitHub Desktop更简单，推荐新手使用

Q: 两个都安装会冲突吗？
A: 不会冲突，可以同时安装使用
