================================================================
🔧 其他解决方法
================================================================

方法A：临时允许PowerShell运行（当前窗口）
----------------------------------------
在PowerShell中执行：

Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

然后再运行：

npm install @supabase/supabase-js


方法B：永久修改策略（需要管理员权限）
----------------------------------------
1. 右键PowerShell，选择"以管理员身份运行"
2. 执行：

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

3. 提示时输入 Y 确认
4. 重新打开普通PowerShell即可

❓ 为什么会出现这个错误？
================================================================

Windows PowerShell 默认不允许运行脚本，这是安全策略。
npm 在 PowerShell 中会调用脚本文件，所以被阻止了。
